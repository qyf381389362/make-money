# Project Backlog & Todos — 待办任务与路线图

本文档用于记录 Make Money 项目当前的版本路线图和未来待开发的高级任务（Backlog）。任务按开发阶段分类，便于开发者快速认领。

---

## 📅 版本路线图 (Roadmap)

### 🟢 v1a — 基础持仓与决策日记 (已发布 ✅)
*目标：建立本地存量记账、日均收盘行情自动刷新和操作决策记录的基础闭环。*
- [x] **T1** — 增加买入均价计算除零保护，防范参数错误导致的进程崩溃。
- [x] **T2** — 为加权平均成本核算、清仓物理删除及超卖保护编写完整的 `pytest` 单元测试。
- [x] **T3** — 在刷新价格网络请求期间，前端增加刷新按钮置灰及 `PortfolioTable` 加载遮罩与骨架屏。
- [x] **T4** — 优化持仓列表行情快照的 SQL 连接查询，利用窗口函数（Window Function）消除 N+1 查询隐患。
- [x] **QA-01** — 修复部分持仓缺乏收盘行情价格时，资产盈亏全显 `--` 的问题（改为均价兜底）。
- [x] **QA-02** — 修复重复刷新价格时由于复合唯一索引引发的 `IntegrityError` 并发回滚崩溃。
- [x] **QA-03** — 修复卖出清仓后持仓记录残留的问题（改为数据库中彻底物理删除）。

### 🟡 v1b — AI 深度决策复盘与基金净值获取 (待开发 ⏳)
*目标：引入本地/云端 LLM 分析能力，并扩展行情渠道以支持多类理财资产。*

#### 1. AI 决策动机问责审计 (AI Audit)
- [ ] **T5 (P1, 预计: 2h)** — 接入 Gemini API / 本地 大模型 API，自动根据用户的“买入/卖出原因”进行投资心理分类（如：跟风、非理性恐慌、逻辑分析、冲动投资）。
- [ ] **T6 (P2, 预计: 1.5h)** — 建立周度 AI 投资审计看板，通过图表展示本周的投资心理偏差（如“由于跟风导致的亏损占比 40%”），提供行为问责警示。

#### 2. 公募基金行情适配 (Fund NAV)
- [ ] **T7 (P1, 预计: 1.5h)** — 扩展 `baostock` 服务层，集成 AKShare 或者天天基金网开放 API，自动拉取场外公募基金的日终最新净值。
- [ ] **T8 (P2, 预计: 1h)** — 前端 `PortfolioTable` 支持展示场外基金和场内 ETF 的混合损益。

#### 3. UX 优化
- [ ] **T9 (P3, 预计: 1h)** — 实现日记按时间段（本周/本月/今年）过滤，增加日记原因的模糊关键字搜索。

### 🔴 v2.0 — 云同步、资产漏斗与自动化导入 (远期规划 🚀)
*目标：从单机走向云端，实现多终端同步、多账户管理和便捷的交割单自动解析。*

- [ ] **T10 (P2)** — **安全认证体系**：引入基于 JWT / OAuth2 的用户账户注册登录，支持多租户数据完全隔离。
- [ ] **T11 (P1)** — **券商交割单自动导入**：支持上传主流券商（如华泰、招商、同花顺等）导出的 CSV 交割单，一键自动解析成 journal_entries 流水并一键初始化持仓。
- [ ] **T12 (P2)** — **大类资产漏斗图**：前端引入可视化库（ECharts/Recharts），向用户展示大类资产（A股、公募基金、场内ETF、现金）的资金占比与配置建议。
- [ ] **T13 (P3)** — **周度/月度损益变化曲线**：增加历史总资产市值变化曲线图，直观展示个人财富的成长轨迹。
